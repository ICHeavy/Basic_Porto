What is a robot?

A robot is a programmable, self-controlled device made up of electronic, electrical, or mechanical units. Essentially, it functions in place of a living agent, often performing tasks that are repetitive, dangerous, or require precision beyond human capabilities. Robots typically have sensors to perceive their environment, actuators to move or manipulate objects, and processors to make decisions based on sensory data.

Real Robots

Robots have appeared in various forms in both fiction and reality. Iconic movie robots include R2-D2 from "Star Wars" and the robot dog from Boston Dynamics. Real-world examples range from industrial robots used in manufacturing to drones and robots designed for space exploration. Additionally, innovative bio-robots are being developed using cells and organic tissue, showcasing the advancing frontier of robotics technology. Unlike traditional desktop computers or phones, which primarily process data and communicate digitally, robots interact with the physical world, using their mechanical and electronic components to perform tasks autonomously.

Autonomous vs Automatic

Autonomous systems can make their own decisions and take action based on their environment, without human input. In contrast, automatic systems follow set instructions and can't adapt to changes or new situations. An autonomous car can navigate city streets and avoid obstacles by itself, while an automatic car wash follows a fixed routine and can't adjust if something unexpected happens.

What’s in a computer?

A computer consists of three main components: the processor, storage, and memory. The processor, or CPU, is the brain of the computer that performs calculations and executes instructions. Storage, like a hard drive or SSD, retains information even when the computer is off, such as saving an essay so you can continue working on it later. Memory, or RAM, temporarily holds data that the computer needs while running, like a variable in a program that only exists while the program is active and is cleared when it stops.

What is a program?

A code, or program, is a set of instructions that a computer executes to perform specific tasks. It tells the computer how to process data, make calculations, and interact with hardware. Programs can range from simple commands to complex systems, enabling everything from displaying text to controlling robots. Writing code involves defining variables, using loops and conditionals, and breaking tasks into functions for efficient problem-solving.

Main Loop and Functions

Code runs sequentially, and when we put code under the "on start program" block, it becomes what we call the main function because the primary part of your code runs there. Functions are blocks of reusable code that achieve a specific purpose, and it’s good practice to name them according to what they do without using spaces. If we create a function, we must call that function under the "on start program" block to use it. Functions need inputs to run correctly, and these inputs must not be empty.

Systems and Functions

A system broadly refers to a set of interconnected components working together for a common purpose. In robotics, this can refer to individual systems on the robot, like the motor system or the sensor system, each performing different tasks. For example, our immune system and nervous system have distinct roles. Similarly, a function we create in programming is also a system because it has inputs and outputs. The data handled in a function are local variables, meaning they exist only when the function is called and are deleted after it's done. Therefore, you can't put something like a counter for your main loop inside a function as it needs to persist across function calls.

Explanation of Categories

  • Controls: Include delays, loops, and if statements.
    • Delays: Pause the program for a certain amount of time.
    • Forever: Loops a block of code forever.
  • Operators: Perform operations on data such as math, random numbers, and data conversion.
    • Math Blocks: All math blocks are considered as a set of parenthesis (e.g., 1 - (2 + 3)).
    • Conversion: Necessary for treating data differently based on its type, including:
      • Booleans: True or false values.
      • Floats: Decimal numbers.
      • Integers: Integers.
      • Strings: Words or arrays of characters, more relevant in scripting.
  • Comparators: Compare values using equality and logical operators (and/or).
  • Sensors: Variables that contain the value of the robot's sensors, which can be set and read.
  • Communications / Events: Topics for a future class.

How to Code

Drag and drop the coding blocks just like in Scratch. Each start block runs once every time you hit play. If you want your robot to keep doing something or repeat actions, you’ll need to code those actions explicitly. All computers follow this principle of running instructions as programmed.

Using Comments

You can right-click to access comments, duplicate blocks, or get help to understand how a block works. Adding comments is crucial for clarity and troubleshooting:

  • What to Comment: Describe what your code is doing (e.g., "This function calculates the missing side of a triangle").
  • Why to Comment: If anything was confusing when you did it or required help to fix, make sure to comment on that as well.

Mathematical Operations

We can do math with all basic operators and even some advanced ones. Each block should be treated with parentheses around it, like (1-(2+3)), to ensure correct order of operations. This helps in understanding and debugging your code more effectively.

By following these guidelines, you'll be able to write clear, effective, and well-documented code for your Sphero RVR projects.

Why Can't You Make It Go Any Faster?

This limitation is due to the clock speed of the computer inside the robot. The clock determines how fast the robot can execute each instruction. Think of it as the computer’s heartbeat—each beat allows one instruction to be processed. Clever programming strings these limited instructions together to perform complex tasks, but each individual operation can only happen at the clock’s speed.

Activity: Advanced Lights and Controls

Objective: Recreate the police car effect without directly selecting the colors red or blue. Experiment with new blocks and techniques.

Background:

LEDs have three color channels: Red, Green, and Blue (RGB). Each channel is an integer value from 0-255, stored as an array of three numbers: [R, G, B]. This scale ranges from 0 (off) to 255 (max). When all channels are at 0, the color is black; when all are at 255, the color is white. Primary colors are red, blue, and yellow, while secondary colors are green, orange, and purple. To isolate primary colors, adjust the channels of secondary colors by removing one channel. Instead of directly using red or blue, create these colors by adjusting the RGB channels. For example, to create red, use an orange color like [255, 165, 0] and reduce the green and blue channels to 0. To create blue, use a purple color like [128, 0, 128] and reduce the red and green channels to 0.

Instructions:

  1. Create and set RGB values to simulate red and blue.
  2. Use the RGB values to set the LEDs to red and blue using indirect color manipulation.
  3. Address the lights using the variables storing the address for the lights found in the operators and sensors sections.
  4. Use a loop to alternate between the two colors.
  5. Add delay blocks to control the flashing speed.
  6. Use a forever loop to make the lights flash continuously.
  7. Experiment with different blocks to achieve the desired effect.

Challenge/Questions:

  • How does changing the RGB values indirectly help in achieving the desired color effect? Do any colors work better than others?
  • Experiment with different blocks and find the most efficient way to create the police car effect.
